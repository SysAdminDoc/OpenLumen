## Hard gates (all must pass)

### Gate 1 тАФ Audit hardening fixes are landed

The 2026-05-17 in-tree audit pass touched 15 Kotlin / test / strings
files. They are present on `main` working tree but **not yet
committed**. Commit them in a single focused commit before cutting
v0.5.0.

```bash
cd Z:/repos/OpenLumen
git status                                # confirm 15 unstaged files
git add app/src/main core-engine/src core-prefs/src core-schedule/src
git diff --cached --stat                  # sanity-check what's staged
git commit -m "Hardening (2026-05-17 audit): solar, polar, mid-ramp, sanitization, overlay cutout"
```

The CHANGELOG `[Unreleased]` "Hardening" subsection (added in rev 4.1)
documents the line-item list.

### Gate 2 тАФ Pass-2 HIGH-severity correctness fixes (C132-C135)

Surface-level race conditions the audit didn't catch. Each is small
(~5-30 lines of Kotlin). Land them in 1-4 commits **before tagging
v0.5.0**.

- **C132** тАФ `LumenService.applyMatrix` ramp-scheduling atomicity.
  Wrap the `transitionJob?.cancel(); join(); transitionJob = launch{}`
  block in a dedicated `rampMutex.withLock { тАж }`. See
  [.ai/research/2026-05-17/SECOND_PASS_FINDINGS.md](../.ai/research/2026-05-17/SECOND_PASS_FINDINGS.md#1-major-findings).
- **C133** тАФ `LumenService.clearAndStop` cancel-and-join
  `transitionJob` before `engine?.clear()`. Trivial.
- **C134** тАФ `ColorDisplayManagerEngine.load` reset `cdm = null` when
  returning null after a partial reflective failure. Trivial.
- **C135** тАФ `OverlayEngine.installView` thread-safety with
  `apply`/`clear`. Add a private `installMutex` to the engine, OR
  route `installView` through `applyMutex` at the service level.

### Gate 5 тАФ F-Droid recipe + buildserver dry run

Refer to [fdroid-recipe-draft.md](fdroid-recipe-draft.md) for the

  ```bash
  fdroid build com.openlumen --on-server   # optional local build dry run
  ```

- Apply the `version-control-info.textproto` reproducibility fix
  (C120) per [reproducible-build.md](reproducible-build.md) and the

### Gate 6 тАФ Versioning + release tag

In `app/build.gradle.kts`:

```kotlin
versionCode = 6
versionName = "0.5.0"
```

Then:

```bash
# Update fastlane changelog for the new version
mv fastlane/metadata/android/en-US/changelogs/{5,6}.txt
# Edit 6.txt to match v0.5.0 (тЙд500 chars)

git commit -am "Bump version to 0.5.0 (versionCode 6)"
git tag v0.5.0
git push origin main v0.5.0
```

The release workflow (`workflow_dispatch` on `.github/workflows/release.yml`)
takes the version as an input, builds, attests, hashes, and uploads to a
GitHub Release.

### Gate 7 тАФ CI green on the release commit

All three workflows must succeed:

- `ci.yml` тАФ `assembleDebug + lint`, module unit tests,
  **`permissions-audit`** (network + location + phone + usage + a11y
  blocklist as of rev 4.1).
- `release.yml` тАФ release APK + no-banned-permissions assertion +
  SHA-256 + build-provenance attestation.
- `sbom.yml` тАФ SPDX SBOM + Anchore advisory scan; non-blocking, but
  any new High / Critical findings must be triaged before merging the
  release commit.

### Gate 8 тАФ F-Droid submission MR (C140)

Once Gates 1тАУ7 pass:

1. Fork `gitlab.com/fdroid/fdroiddata` to your GitLab account.
2. Add `metadata/com.openlumen.yml` per
   [fdroid-recipe-draft.md](fdroid-recipe-draft.md).
3. Open MR titled `New app: com.openlumen (OpenLumen)`.
4. Wait 24тАУ48h post-merge for the repo to refresh.

## Pre-flight commands (run all of these on `main` before tagging)

```bash
cd Z:/repos/OpenLumen

# 1. Build artefacts
./gradlew clean
./gradlew :app:assembleDebug :app:lint --stacktrace

# 2. Tests
./gradlew \
  :app:testDebugUnitTest \
  :core-engine:test \
  :core-schedule:test \
  :core-prefs:test \
  --stacktrace

# 3. Manifest + classpath audit (matches the permissions-audit CI job)
./gradlew :app:assembleDebug
APK=$(ls app/build/outputs/apk/debug/app-debug.apk)
AAPT=$ANDROID_HOME/build-tools/$(ls -1 $ANDROID_HOME/build-tools | tail -n1)/aapt2
"$AAPT" dump permissions "$APK" | \
  grep -E "android\\.permission\\.(INTERNET|ACCESS_NETWORK_STATE|ACCESS_WIFI_STATE|ACCESS_FINE_LOCATION|ACCESS_COARSE_LOCATION|ACCESS_BACKGROUND_LOCATION|READ_PHONE_STATE|QUERY_ALL_PACKAGES|PACKAGE_USAGE_STATS|BIND_ACCESSIBILITY_SERVICE)" \
  && echo "FAIL: banned permission in merged manifest" || echo "OK"

./gradlew :app:dependencies --configuration releaseRuntimeClasspath > /tmp/deps.txt
grep -E "(play-services|firebase|com\\.google\\.android\\.gms)" /tmp/deps.txt \
  && echo "FAIL: disallowed dependency" || echo "OK"

# 4. Release APK (requires keystore env vars)
./gradlew :app:assembleRelease --stacktrace
SRC=$(ls app/build/outputs/apk/release/app-release*.apk | head -n 1)
"$AAPT" dump permissions "$SRC" | grep INTERNET && echo "FAIL" || echo "OK"
sha256sum "$SRC"
```
